## Opportunity

The 2026-07-13 plan note (recovered from the pre-sync WIP stash) recorded an intent to iterate lappe-linter toward a first-class product-intent offering rather than a personal-config Obsidian linter fork. Framing the plugin as a product changes what the settings surface, scope model, and docs need to be, so the direction should be scoped before more feature work lands on the current plan of record.

## Description

Scope what "product-intent offering" means for lappe-linter:

- Define the target user beyond the author (who installs this and why).
- Decide how a product framing reconciles with the current `cross-project-plugin-plan-of-record` and the 2026-07-13 settings delivery plan already merged to master.
- Identify what the redesigned settings surface (merged in PR #6/#8: `lappe-preview-view.ts`, canonical settings tabs, scope inheritance) already delivers toward that intent versus what is still personal-config shaped.

Origin: stashed plan-doc note "Pivoting direction to iterate towards a first class product-intent offering." This backlog item captures only the strategic intent.
