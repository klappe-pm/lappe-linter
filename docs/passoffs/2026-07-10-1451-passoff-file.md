## Lessons learned

- Rigor: this was product/UI work that warranted verification in the real app; it was treated as if unit tests were sufficient. The calibration takeaway: UI/product changes are not done until observed working in the target surface (Obsidian), never from a green test suite alone.
- Root cause of the wasted session: the spec was never restated in the plugin's own user-facing nouns ("Headers tab", "Body tab") and diffed against what was being built, so the whole redesign landed in the wrong place (a "Lappe" tab) and downgrades (side-by-side to modal, autocomplete to free-text) were marked done. Prevention: restate the spec in target nouns before building, ship the smallest visible slice first and confirm it renders, and never mark a downgraded item done.
