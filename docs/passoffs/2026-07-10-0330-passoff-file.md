## Architecture of record

Per [[decisions]] dec-004 the Obsidian plugin stays at the repo root on upstream layout (keeps `git merge upstream/master` cheap) and all new work lives in two pure workspaces: `packages/core` (`@lappe-linter/core`, zero obsidian imports, enforced by a test) and `packages/cli` (`@lappe-linter/cli`, bin `lappe-linter`). Core exposes one entry, `lintText`, composing config load, provider merge, scope resolution, and the rule runner; the CLI and the future plugin wiring both call it, so parity holds by construction. Contract types live in `packages/core/src/config/types.ts`.
