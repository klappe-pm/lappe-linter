## Lessons learned

- Rigor: high (ad-hoc session, no scope anchor since it did not start with /resume). Obligations met: independent review ran as a three-auditor workflow round with adversarial verification of security findings before any fix landed; full suite green at every landing; end-to-end CLI smoke performed by the session itself rather than trusting agent claims.
- Lesson: subagent capacity is a shared session budget. Why: the final feature and verify agents died on the session token limit mid-workflow, and the work had to finish inline. How to apply: sequence heavy multi-agent rounds earlier in a session, keep a solo-completion fallback for the tail, and treat agents-unavailable as an expected mode, not an error. Recurrence: first. Enforced-by: not yet.
- Lesson: verify smoke tests against a config that omits defaults.rules, not only the scaffold. Why: the code-checks carrier rules silently never ran under a minimal user linter.yaml; only the session's own hand-driven smoke caught it. How to apply: every new rule family gets a minimal-config activation test in the same change. Recurrence: first. Enforced-by: the new lintText enablement test in packages/core/__tests__/code-checks/code-checks.test.ts.
