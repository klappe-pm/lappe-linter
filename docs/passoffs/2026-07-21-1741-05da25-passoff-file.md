## Lessons learned

Rigor: high was warranted (multi-file plugin surface, merged PRs, redeploy to 12 vaults), but obligations were NOT met and the session is a confirmed rigor mismatch. The core failure: I diagnosed from git/code state and built on my own interpretation instead of finding and confirming the actual requirements first. The user had authored a detailed UI spec on 2026-07-10 that lived in a sibling session's transcript; I never looked for it and instead re-built the precise "bolted-on Lappe tab / standalone view" antipattern the user had already rejected across multiple past sessions ("stop fucking making lappe tab"). Calibration takeaways for the next session: (1) for a project with a `docs/plans/` tree, READ the plans before writing any code; they encode hard non-goals. (2) When a user says "this is not what I asked for" more than once, stop implementing and go find the original requirements; do not keep guessing the symptom. (3) Confirm intent against a durable spec, not against the code's current shape. Recovered spec now lives at `docs/lappe-linter-ui-requirements-recovered.md` so this cannot recur.
