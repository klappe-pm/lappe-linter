## Key decisions

- `aliases`/`tags` stay pinned last, not part of the reorderable 8-key `priority-keys`; listing them would push unlisted keys (e.g. `status`) below them.
- The lappe core is the single engine; kept upstream YAML/heading/body rules are surfaced in the Lappe tab and default on (opt-out); yaml-title/title-alias/footnote stay retired.
- Backlink/alias scope matchers read a plugin-supplied MatchContext (metadata cache); the CLI passes none, preserving CLI/plugin parity.
- Disabled `valid-jsdoc` (deprecated, redundant with TS) and added `node` env rather than annotating hundreds of functions; excluded generated `dist/` from lint.
- Timestamp keys are `date-created`/`date-revised` (the request's `date-updated` reading).

## Risks and open questions

- Live preview (4.2) is a snapshot modal, not a non-blocking side-by-side leaf; scope-builder fields use free-text, not vault-value autocomplete; per-option profile override editor is basic. All recorded as follow-ups in the plan doc.
- The jobs vault has its own strict frontmatter schema; running Lappe Linter's default rules there WILL rewrite frontmatter and could clash. Test on `_archive/` fixtures, not real job notes.
- Multi-project sync is incomplete by design (see Next actions): only lappe-linter is synced. Others are concurrency-locked, have uncommitted WIP, are diverged, or have no remote.

## Lessons learned

- Rigor: high (multi-file shared plugin surface, config/loader/runner contract changes). Obligations met: per-phase checkpoints recorded in the plan doc, full suite green at every phase, adversarial idempotency tests on each new rule caught a real bug (list-style double trailing newline).
- "Sync all projects" is not a safe bulk sweep in a multi-session, mixed-remote workspace; the concurrency and branch guards exist precisely to prevent cross-session clobbering. Survey first, act per-repo, never blind-commit others' WIP.
